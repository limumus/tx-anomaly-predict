闪电贷攻击深度分析

针对LAURA/WETH流动性池的价格操纵攻击技术解析

1

攻击意图

获取巨额操作资金（30,000 WETH，约9000万美元），用于后续操纵市场。这是闪电贷攻击的标准起始操作，允许攻击者无需自有资金即可进行大规模操作。

4

                  "internal_calls": [
                    {
                      "type": "EVENT",
                      "function": "Sync",
                      "args": "reserve0=4,334,828,289,544,710,831,315, reserve1=23,068,964,517,278,041,866,222"
                    }

                }
              ]
            }
          ]
        }
攻击意图

利用代币合约后门函数和DELEGATECALL执行恶意逻辑，强制修改Uniswap池的储备金数据。此步骤将LAURA储备从17.44e18修改为4.33e18（减少75%），而WETH储备保持不变，制造了虚假价格偏差。

恶意函数调用链

1

LAURA.removeLiquidityWhenKIncreases

攻击者调用LAURA代币合约中的可疑函数，作为攻击的起始点。

攻击入口点
2

DELEGATECALL PumpToken.removeLiquidityWhenKIncreases

通过DELEGATECALL操作码执行PumpToken合约中的恶意逻辑，绕过权限检查。

关键攻击手段
3

UniswapPool.sync

恶意调用Uniswap流动性池的sync函数，强制更新储备数据。

储备修改操作
4

修改reserve0=4.33e18

将LAURA储备从17.44e18修改为4.33e18（减少75%），同时保持WETH储备不变，制造虚假价格。

攻击成功

套利路径闭环

  1. 闪电贷借入 - 30,000 WETH作为攻击资金
  2. 首次兑换 - 用11,526 WETH买入LAURA拉高价格
  3. 添加流动性 - 注入流动性获取LP凭证
  4. 恶意函数调用 - 修改储备金制造价格偏差
  5. 移除流动性 - 利用虚假价格获得超额资产
  6. 卖出LAURA - 兑换回WETH实现套利
  7. 归还闪电贷 - 归还初始借入资金
  8. 利润提取 - 最终获利12.34 ETH

安全漏洞根源

  • 代币合约后门

    removeLiquidityWhenKIncreases函数暴露危险功能，允许恶意操作，这是攻击的主要入口点。

  • Uniswap池权限缺陷

    允许外部合约直接调用sync()修改储备，缺乏必要的权限验证机制。

  • 闪电贷监控缺失

    未检测大额贷款后的异常储备变动，使攻击者能够利用价格偏差套利。
